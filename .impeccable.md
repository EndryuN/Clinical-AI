# Impeccable Design Context — MDT Data Extractor

## Design Context

### Users
NHS clinicians (consultants, registrars, MDT coordinators) reviewing extracted patient data from MDT outcome proformas. They work in time-pressured clinical environments — often reviewing 50+ patients in a session. The tool replaces hours of manual data entry. They need to trust the extraction, quickly spot errors, and move on.

### Brand Personality
**Clinical. Trustworthy. Efficient.**

The tool should feel like a precision instrument, not a consumer app. Every pixel should serve the clinician's workflow. No decoration for decoration's sake.

### Emotional Goals
- "This saves me hours" — speed and efficiency are paramount
- "I can trust this" — confidence through transparency (provenance, source links, G049 citations)
- "Nothing slipped through" — coverage stats and unused text highlighting catch gaps

### Aesthetic Direction
- **Visual tone:** GitHub/Linear-inspired — clean data tables, subtle status indicators, restrained colour palette
- **Theme:** Dark mode default (clinical environments are often dimly lit). Light mode available via toggle.
- **Colour system:** Confidence colours are the primary semantic palette:
  - Green (#C6EFCE / #198754) — structured_verbatim (high confidence)
  - Orange (#FFEB9C / #f97316) — freeform_verbatim (medium confidence)
  - Red (#FFC7CE / #dc3545) — freeform_inferred (low confidence)
  - Grey (#D9D9D9) — edited by clinician
- **Anti-references:** Avoid flashy gradients, rounded bubbly UI, consumer health app aesthetics. This is not a patient-facing tool — it's for clinical professionals.

### Design Principles

1. **Data density over whitespace** — Clinicians process 50+ patients. Compact tables, minimal padding, no wasted space. Information-rich screens.
2. **Colour means something** — Every colour communicates confidence level or status. No decorative colour. Green/orange/red are reserved exclusively for the confidence system.
3. **Progressive disclosure** — Show summary first (confidence badges, coverage %), details on demand (source cell highlight, reason tooltip, coverage overlay).
4. **Source transparency** — Every extracted value must be traceable. Source links, G049 citations, verbatim/inferred indicators are not optional features — they're core UI.
5. **Zero-friction review** — Tab through patients, scan colours, edit inline, move on. No modal dialogs, no confirmation screens for routine actions.

### Typography
- System font stack (Bootstrap 5 default) — fast loading, familiar
- Monospace for timers, patient IDs, and technical values
- 11-13px body text — compact but readable on desktop

### Spacing
- Tight padding (6-8px cells, 12-16px sections) — maximise data density
- Clear visual hierarchy through borders and subtle background differences, not large gaps
